Examen Git y Github

Alumno: Antonio

  1. ¿Que es un sistema de control de versiones?

    Es una herramienta que nos deja guardar y gestionar los cambios que se hacen en los archivos de un proyecto .Sirve para ver qué se modifico, cuando y quien lo hizo. También permite volver a versiones anteriores.

  2. ¿Cual es la diferencia entre git y github?

    Git es el sistema de control de versiones que se usa en local para gestionar los cambios del proyecto. GitHub es una plataforma online donde se almacenan repositorios Git.

  3. ¿Cómo creo una rama en git?

    Usando el comando `git checkout -b nombre_de_la_rama`

  4. ¿Cómo clono un repositorio?

    Con el comando `git clone url_del_repositorio`

  5. ¿Cómo puedo bajar los ultimos cambios de un repositorio?

    Usando `git pull origin nombre_de_la_rama`

  6. ¿Cómo puedo gener un tag?

    Con el comando `git tag nombre_del_tag`

  7. Cómo hago una mezcla de ramas?

    Con el comando `git merge nombre_de_la_rama`

  8. ¿Cómo puedo moverme a un commit especifico?

    Primero localizas el ID del commit con `git log`, luego haces:`git checkout ID_del_commit`

  9. ¿Cómo puedo borrar una rama?

    Si es una rama local: `git branch -d nombre_de_la_rama` Si es una rama remota: `git push origin --delete nombre_de_la_rama`

  10. ¿Cómo puedo borrar un tag?

    Para borrar un tag local: `git tag -d nombre_del_tag` Para borrarlo también en GitHub: `git push origin --delete nombre_del_tag`

  11. ¿Cómo puedo ver los cambios que estan por enviarse al repositorio?

    Con el comando `git status`

  12. ¿Para que sirve el comando git remote -v?

    Muestra las direcciones url asociadas al repositorio remoto. Sirve para verificar a que repositorio estas conectado y desde donde se hara el push o pull

  13. ¿Cómo le digo a git cual es mi email y correo?

    git config --global user.name "Antonio" git config --global user.email "antonio@email.com"

  14. ¿Cuales son los pasos para conectar git con github?

    1. Crear un repositorio nuevo en GitHub
    2. Copiar la URL del repositorio
    3. En tu proyecto local, inicializar Git con git init
    4. Conectar el repositorio remoto con git remote add origin url
    5. Confirmar con git remote -v
    6. Subir los archivos con git push -u origin main

  15. ¿Cómo puedo crear un alias en git?

    Se crea con el comando git config --global alias.nombre "comando"
